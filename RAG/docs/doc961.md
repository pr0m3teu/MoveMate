## Description

Enable the Poseidon hash function on mainnet.

There is a small change made to the function as well: It now only accepts between 1 and 16 inputs. Previously, larger inputs were hashed using an 16-ary Merkle tree which is compatible with zkLogin but not other Poseiodon implementations which typically uses a 2- or 4-ary Merkle tree.

This new version follows what seems to be the consensus on how to implement Poseidon over BN254, and the limit of 16 inputs in the new version is compatible with <https://github.com/iden3/go-iden3-crypto/tree/master/poseidon>, <https://github.com/iden3/circomlib/blob/master/circuits/poseidon_constants.circom> and <https://github.com/arnaucube/poseidon-rs>. We leave it to the user to decide how to handle larger inputs but provide an example (in form of a unit test) of how to it for an 16-ary Merkle tree as used in zkLogin.

See also the SIP, <https://forums.sui.io/t/sip-12-poseidon-in-sui-move/45136>.
